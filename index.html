1. 사업 개요 (Service Overview)

1.1 서비스 비전 및 미션

Vision

매장과 고객을 데이터로 연결하여, 오프라인 F&B 비즈니스의 디지털 전환(DX)을 선도하는 글로벌 No.1 SaaS 플랫폼

Mission

오프라인 매장의 운영 비용을 혁신적으로 절감하고, 초개인화된 맞춤형 주문 경험을 통해 매출 극대화와 고객 만족을 실현

1.2 시장의 문제점 및 한계

  • 높은 인건비 및 구인난: 국내 F&B 고정비 중 인건비 비중은 30% 상회, 구인난으로 매장 운영 난이도 극대화.
  • 하드웨어 도입/유지보수 비용: 테이블당 과도한 초기 설치비(30~50만 원) 및 렌탈료 부담, AS 대기로 인한 주문 공백 및 전력 소모.
  • 데이터 파편화: POS, 배달 앱, 테이블오더가 분리되어 고객 행동 데이터(주문 주기, 탐색 패턴 등) 마케팅 활용 불가.

1.3 QR 주문 서비스의 필요성 (Solution)

Zero-Hardware

단말기 구매나 배선 공사 없이 QR 코드 스티커 부착만으로 즉시 도입.

비대면 즉시성

앱 다운로드 없이 스마트폰 카메라 스캔 3초 만에 메뉴 조회 및 결제 완수.

효율성 극대화

테이블 회전율 25% 증가 및 홀 서빙 리소스 40% 절감.

주요 DDL 스키마 (데이터베이스 설계)

-- 1. Orders (주문 정보 마스터 테이블)
CREATE TABLE Orders (
    order_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    store_id UUID NOT NULL REFERENCES Stores(store_id) ON DELETE CASCADE,
    table_no VARCHAR(20) NOT NULL,
    order_status VARCHAR(30) NOT NULL DEFAULT 'PENDING', 
    total_amount DECIMAL(12, 2) NOT NULL DEFAULT 0.00,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 2. Payments (결제 승인 내역 테이블)
CREATE TABLE Payments (
    payment_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id UUID NOT NULL UNIQUE REFERENCES Orders(order_id),
    pg_transaction_id VARCHAR(100) UNIQUE,
    payment_method VARCHAR(50) NOT NULL,
    payment_status VARCHAR(30) NOT NULL DEFAULT 'READY',
    paid_amount DECIMAL(12, 2) NOT NULL
);
